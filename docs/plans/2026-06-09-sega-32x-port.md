# Plan: Sega 32X subsystem (staged port)

**Date:** 2026-06-09
**ADR:** [0011-sega-32x-subsystem](../adr/0011-sega-32x-subsystem.md)
**Pattern:** mirrors the Sega CD staged port (`src/manifests/segacd/NOTES.md`,
phases A–D) and the m68000's phased build-out.

## Goal

A functional Sega 32X layered additively on the Genesis (M8 family): two SH-2
CPUs, the 32X VDP composited with the Genesis VDP, PWM audio mixed with the
Genesis sound, and the shared comm/bank/interrupt register file — driven from
the player via a `sega32x` family + `.32x` routing. A plain Genesis must never
traverse any 32X path (byte-parity floor untouched).

## Hard prerequisites (blockers, surfaced up front)

These are not implementation steps; they gate *validation* and *cannibalization*
and are outside the container today:

1. **Emu reference tree.** The goal says to cannibalize from
   `C:\Users\mkrol\source\repos\Emu`. That tree is a local Windows path and is
   **not present in this cloud container** — nothing can be lifted from it here.
   To reuse Emu's 32X/SH-2 code it must be made reachable (vendored into the
   repo, added as a submodule, or the relevant sources pasted in). Otherwise the
   SH-2 + 32X are written from public hardware documentation (the route this
   plan assumes).
2. **32X BIOS / security (vector) ROM + commercial ROM images.** "100% parity"
   is only *verifiable* against real software. The repo never commits these;
   the boot/parity/conformance tests skip without them (as the Genesis and Sega
   CD suites already do). Functional development can proceed without them, but
   the parity claim cannot be signed off until they are available at validation
   time.
3. **SH-2 conformance corpus** (optional but recommended), wired the same way
   as the m68000 corpus (`SKIP_RETURN_CODE 4`, env-var-pointed, never
   committed).

## Phase A — SH-2 CPU core (`src/chips/cpu/sh2`)

The reusable, system-agnostic piece. Buildable and testable with **no** 32X,
no Emu, and no ROMs — only public SH-2 ISA documentation.

- **A1 (this increment):** chip skeleton — `icpu` subclass, the SH7604
  programming model (R0–R15, PC, PR, SR with T/S/I0–I3/Q/M, GBR, VBR, MACH/MACL),
  big-endian `ibus` 16/32-bit assembly helpers, `reset` (PC/SP loaded from the
  vector table at the reset vector), `tick`/`step_instruction` plumbing,
  save/load state, `introspection_surface` (register_view + trace_target),
  factory registration (`hitachi.sh7604`), and a power-on/identity/state test.
  Undecoded opcodes execute as documented no-ops initially (the m68000's
  bring-up convention).
- **A2:** data-transfer + ALU instruction groups (MOV variants and addressing
  modes, ADD/SUB/CMP/MUL/DIV-step, logicals, shifts/rotates), with per-group
  unit tests.
- **A3:** control flow + system (BRA/BSR/BT/BF/JMP/JSR/RTS/RTE, T-bit ops,
  TRAPA, the exception/interrupt model), then the on-chip peripherals the 32X
  uses (FRT/WDT timer, DMAC, INTC, serial/comm) trimmed to the 32X subset.
- **A4:** SH-2 conformance harness (skips without corpus).

## Phase B — 32X system core (`src/manifests/sega32x`)

- Two SH-2 instances (master/slave) on a `topology::bus` mapping: the 32X SDRAM,
  the cartridge ROM bank window, the SH-2 on-chip register region, and the
  shared **system register file** (comm ports `$20–$2F`, interrupt control/clear,
  the 68000↔SH-2 handshake/adapter-control/reset bits, bank register).
- Framebuffer-access arbitration bits (which side owns the frame buffer) and the
  SH-2 interrupt sources (V/H/cmd/PWM) — state only at this phase; dispatch wired
  with the scheduler in phase D.

## Phase C — 32X VDP + PWM

- **32X VDP:** 256 KB frame buffer (two banks + line table), packed-pixel (8 bpp
  → 256-entry CRAM) and direct-color (15 bpp) modes, the auto-fill register
  block, and `ivideo` output. Implemented as a manifest-tier component.
- **Video composition:** per-pixel priority compositing of the Genesis VDP
  output with the 32X VDP output into the combined `ivideo` presented to the
  runtime. Genesis VDP unmodified.
- **PWM:** stereo PWM source clocked off the SH-2 timer, producing samples mixed
  with the Genesis YM2612 + SN76489. (The audio-extraction capabilities from the
  asset-extraction work apply here for free once it emits samples.)

## Phase D — Genesis-bus bridge + scheduling + player

- Main-side `$A130xx`/`$A15xxx` windows (system registers + Genesis-side
  framebuffer/palette access), the `$000000` security/vector-ROM + cartridge
  remap, and the 68000↔SH-2 comm/handshake bridge.
- SH-2 ↔ 68000 scheduling (clock ratios, the framebuffer-ownership stalls).
- Player adapter `src/apps/player/adapters/sega32x/` + the `sega32x` family +
  `.32x` ROM routing (and security-ROM handling). Additive: a plain Genesis
  never traverses any of it.

## Validation strategy

- Unit tests per phase (no ROMs needed) — the SH-2 ISA groups, the system
  register semantics, the VDP modes, the PWM sample shape, the composition.
- Boot/parity tests behind env-var-pointed ROM/BIOS dirs that **skip** when
  unset (Genesis/Sega CD convention), so CI stays green without the artifacts
  and parity can be checked locally once they are supplied.
- SH-2 conformance test (skips without corpus).

## Status

- Phase A1: SH-2 chip skeleton — **done** (merged).
- Phase A2: SH-2 instruction set — **done**. Cannibalized from the Emu reference
  with per-group unit tests: data transfer, the ALU group (ADD/ADDC/ADDV,
  SUB/SUBC/SUBV, NEG/NEGC, the CMP variants, DT, EXTU/EXTS), logicals, shifts and
  rotates, multiply (MUL.L, MULU/MULS.W, DMULU/DMULS.L, MAC.L/MAC.W with SR.S
  saturation), the divide-step group (DIV0U/DIV0S/DIV1), and TAS.B.
- Phase A3: control flow + system registers + addressing — **done**. The
  delayed-branch mechanism (the delay-slot instruction runs before control
  transfers) + BRA/BSR/BRAF/BSRF/BT/BF/BT-S/BF-S/JMP/JSR/RTS; the LDS/STS and
  LDC/STC system-register transfers (incl. the `.L` stack forms, SR masked); and
  the remaining addressing modes (displacement, GBR-relative, R0-indexed, MOVA,
  immediate logical).
- Phase A3 exceptions/interrupts — **done (core)**. TRAPA + RTE, and the external
  interrupt path: an SH-2-specific `set_irq(level, vector)` + accept callback (no
  `icpu` widening), acceptance at the instruction boundary when level > SR.IMASK
  (push SR+PC, vector through VBR + vector*4, raise IMASK), and a one-instruction
  interrupt inhibit after SR loads. Designed with a Codex consult; the Saturn INTC
  queue/ranking is intentionally NOT ported (the 32X glue owns source latching /
  masks / priority / redelivery).
- Phase A3 SLEEP + illegal-instruction — **done**. SLEEP halts the CPU until an
  interrupt resumes it; any undecoded encoding (including the absent SH7604 FPU
  opcodes) raises the illegal-instruction exception — general-illegal (vector 4)
  normally, slot-illegal (vector 6, saving the branch's resume target) inside a
  delay slot, with `branch_delayed` refactored to yield PC when the slot vectors.
  **This completes the SH-2 (SH7604) CPU's functional model.** Only the
  address-error exception remains deferred (it needs misaligned-access fault
  plumbing) and instruction timing (deferred per ADR-0011).
- Phase B — **in progress** (design from a Codex consult). The on-chip
  peripherals live with the SH-2 chip as a `sh2_peripherals` member (composition);
  the `$FFFFFE00` window is intercepted in `sh2::rd8/wr8` before the external bus
  so the two cores never alias. The 32X's VINT/HINT/CMD/PWM stay *external* (the
  manifest will present them via `set_irq`), so the on-chip INTC is only needed
  once FRT/WDT/DMAC interrupts do.
  - Increment 1 (the peripheral register-window shell + intercept) — **done**.
  - Increment 2 (the `sega32x_system` manifest skeleton) — **done**. Two SH-2s on
    two CPU-local buses (`master_bus`/`slave_bus`) sharing the 256 KB SDRAM
    (`$06000000`), the frame buffer (`$04000000`), and the 8-word COMM bank
    (`$40000020`), each booting from its own ROM at `$0`; plus the reset
    hold/release. Hand-wired like `segacd_system` (`assemble_sega32x`).
  - Increment 3 (the per-CPU IRQ masks/latches mechanism) — **done**. The four
    32X sources (VINT/HINT/CMD/PWM) latch on both CPUs regardless of mask;
    `raise_*` delivers to each CPU whose mask enables it and whose pending level
    it outranks; `set_*_irq_mask` and the SH-2 IRQ-accept callback re-deliver a
    still-latched edge (the Mars flip-flop semantics). Ported from the Emu
    reference, with the SH-2 `pending_irq_level()`/`pending_irq_vector()`
    accessors the arbiter needs.
  - Increment 3b (the SH-2-side system-register window) — **done**. Mapped at
    `$00004000` (+ the `$20004000` cache-through mirror) on both buses: adapter
    control (`$00/$01`, the Mars even=low/odd=high byte-lane quirk), the
    self-referential interrupt-enable register (`$03` sets the executing CPU's own
    mask via `set_*_irq_mask`), and the COMM bank moved to its real `$00004020`
    offset. PWM/VDP/DMA-FIFO registers are stubbed.
  - Increment 4 (the on-chip FRT) — **done**. The SH7604 free-running timer
    (`$FE10-$FE17`) in `sh2_peripherals`: a 16-bit counter clocked off the
    TCR-selected prescale (φ/8·32·128), matching OCRA/OCRB and wrapping, with the
    FTCSR status flags (OVF/OCFA/OCFB, write-0-to-clear-after-read) and the
    compare-clear cadence. Ticked from `sh2::step_instruction` (including during
    SLEEP). Pollable; the timer interrupts (gated by TIER) land with the on-chip
    INTC. Input capture is not modelled.
  - Remaining: the on-chip INTC (vector resolution + delivering the FRT/WDT/DMAC
    interrupts), the WDT and DMAC, the SH-2 conformance harness, the 32X VDP +
    PWM (phase C), and the Genesis-bus bridge + scheduling + player
    (`sega32x_machine`, phase D).
