# rf5c68 — Ricoh RF5C164 PCM (Sega CD)

`mnemos::chips::audio::rf5c68` — the Sega CD / Mega-CD PCM synthesiser. Eight
voices play 8-bit sign-magnitude samples from a shared 64 KB wave RAM, each with
envelope (volume), pan, 16-bit frequency divider, loop-start and start address.
A `0xFF` sample byte is the loop/stop sentinel. Native output ~32.55 kHz stereo.

## Provenance

Ported from the Emu reference `chips/rf5c68` (our own earlier project —
sanctioned per ADR-0006 / [[mnemos-port-emu-cores]]); clean-room per the Sega
Mega-CD Hardware Manual + RF5C68 datasheet. No third-party emulator source
consulted. Restructured to the Mnemos `iaudio_synth` chip contract (metadata /
tick / reset / save_state / introspection / factory registration) and the
sn76489 audio-capture model (`tick()` queues interleaved L/R, drained via
`drain_samples`).

The CTRL register decode follows the datasheet MOD-bit model: bit 7 = enable,
bit 6 = MOD selecting what bits 3:0 carry -- MOD=1 picks the current channel
(bits 2:0), MOD=0 picks the wave-RAM bank (bits 3:0). One write sets one or
the other, never both; real Sega CD drivers select voices with `0xC0 | ch`,
which a combined decode mis-routed to voice 4 for every channel.

## Conformance

`tests/rf5c68_test.cpp`:
- **PCM golden** — sample-for-sample equality with the reference over a
  deterministic scenario (sign-magnitude waveform, fractional pitch, stereo pan,
  `0xFF` loop). Golden generated by compiling+running the reference chip.
- Register read-back + CTRL/CHAN decode + disabled-chip silence.
- `save_state`/`load_state` bit-identical round-trip.

## Sega CD wiring (later, phase C)

Driven by the sub-CPU: register window + bank-selected 4 KB wave-RAM window on
the sub-CPU bus; CDC PCM-DMA blits sectors into wave RAM via the `waveram()`
span. `set_clock_divider()` will be set to the chip's input-clock / native-rate
ratio when the Sega CD scheduler is wired.
